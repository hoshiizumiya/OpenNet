## 概览

我们的当前使用框架为 WinUI3 C++/WinRT。使用 WinUI3 可以让我们创建现代化的 Windows 应用程序，它有更加现代的外观，使用 C++/WinRT 能够提供更好的用户体验和性能。  
目前我们的计划仅在 Windows 上运行，未来可能会考虑跨平台支持。(使用 MAUI 构建系统原生框架的跨平台应用，采用 P/Invoke 跨平台调用)  
目前主要使用的开源库为 libtorrent 。

## 如何构建调试项目

1. 开发环境要求

* Windows 11 24H2 或更高版本，我们建议使用最新版本的 Windows 11。
* Visual Studio 2022 17.14 最新版。（对于 64GB 以上运行内存的机器我们推荐 VS 2026，能够更好地支持大型项目）
* C++ 桌面开发工作负载、WinUI 桌面开发工作负载、C++ WinUI 应用开发工具。
* Windows 11 SDK (10.0.26100.0 或更高版本)
* MSVC v143 (VS 2026 可使用 v145)
* vcpkg 包管理器(VS自带的即可)

2. 构建项目

* 打开 Visual Studio，选择克隆存储库。
* 输入项目的 Git 仓库地址，选择一个大于 30GB 剩余空间的本地路径，路径中不要存在任何空格和非英文字符，一些旧依赖会因此报错，然后点击 "克隆"。若网络不佳，建议使用 SSH 克隆或 TUN 代理 Git。
* 在解决方案资源管理器中，确认加粗字为 OpenNet 即配置启动项目。
* 可以直接选中深绿色按钮开始调试以启动应用程序。
* 初次运行时，应用程序会自动下载所需的 nuget 和 vcpkg 依赖库文件并进行编译，取决于不同配置可能需要一些时间（30-60m左右），请耐心等待。由于 vcpkg 由 github 托管，若网络受限建议使用代理工具。

## 功能

* 支持 BitTorrent 协议的种子文件和磁力链接下载。

## 开发路线图

* 支持 HTTP/HTTPS/FTP 等协议的文件下载。
* 支持自动 NAT 穿透。
* 支持 DHT、PEX、LSD 等分布式网络功能。
* 支持 NAT 检测。
* 支持远程控制。
* 支持用户登录和多用户管理。
